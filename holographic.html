Holographic Email Notifier

An ambient home device that surfaces priority email alerts through a holographic display — keeping you aware without forcing you onto a screen.

/ The Problem

Email overload at home is a quiet but persistent friction. You either keep your phone or laptop nearby — and get pulled into a notification spiral — or you put them away and miss something urgent.

We wanted to design a non-intrusive way to surface priority email alerts while still letting the home stay a home. The device needed to:

  • Be ambient enough to disappear when nothing is happening
  • Surface only high-priority emails, not every notification
  • Not require glancing at a phone or screen
  • Feel like a piece of furniture, not a gadget
/ The Approach

We chose to display information using the Pepper's Ghost effect — a 19th-century optical illusion that uses an angled transparent surface to reflect an image from below, creating the appearance of a floating hologram.

SKETCH / DIAGRAM How Pepper's Ghost works in our setup
BUILD PHOTO Internals — Raspberry Pi + screen + prism

In its default state, the device displays a holographic clock — ambient, glanceable, decorative. When a priority email arrives, the display switches to a brief notification overlay that the user can dismiss or attend to.

System architecture

  • Display layer: Small screen pointing upward at a 35° transparent prism
  • Compute: Raspberry Pi 4 (4GB) running Python notification logic
  • Email integration: APIs to fetch and filter incoming messages
  • Priority logic: Sender-based and keyword-based filtering
/ Key Decisions

The 35° prism angle

We tested multiple prism angles to find the right balance between viewing comfort, image brightness, and physical footprint. 35° proved to be the sweet spot — steep enough for a clear floating image, shallow enough to keep the device compact and stable on a shelf.

ANGLE TEST PHOTOS Side-by-side of different prism angles

Filter aggressively, not gently

Early prototypes surfaced too many emails — defeating the purpose. We moved to an aggressive filtering model: by default, nothing is shown. Only emails matching specific high-priority criteria break through. The ambient clock stays ambient.

Build the optics before the software

We deliberately got the physical optics working before writing any backend logic. If the Pepper's Ghost effect didn't look good, no amount of clever code would save the product. This kept us honest about whether the core concept was viable.

/ Results & Testing

We validated the design through iterative testing on:

  • Viewing angle: Image legibility across a 0–60° viewing cone
  • Brightness: Performance under various ambient room lighting
  • Notification clarity: Whether the alert was readable at a glance from across the room
  • Ambient unobtrusiveness: Whether the idle state genuinely receded into the background

/ What I'd Do Differently

Three things I'd change on a v2:

  • On-device ML for priority detection. Right now filtering is rule-based — a lightweight ML model could learn what counts as "urgent" for each user without manual setup.
  • Custom prism mount. The current setup uses a generic prism in a 3D-printed cradle. A custom-machined mount would tighten tolerances and improve image clarity.
  • Reduce idle power draw. The Pi 4 is overkill for a device that mostly displays a clock. A Pi Zero or microcontroller with a separate display driver would cut power consumption substantially.
