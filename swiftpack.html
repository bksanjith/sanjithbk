SwiftPack — E-commerce Order Packing Terminal

A Raspberry Pi-based packing station that aggregates orders from Shopee, Lazada, and Shopify into a single workflow — targeting a 70% reduction in per-parcel packing time.

/ The Problem

Running a multi-platform e-commerce business means context-switching between Shopee Seller Centre, Lazada Seller Centre, and Shopify constantly. Every order has its own login, its own label format, its own quirks.

At Fragrance Castle, I watched this process eat into the day. The pain wasn't packing — it was everything around packing:

  • Logging into separate seller portals for every batch
  • Navigating to the right order, generating the label, downloading the PDF
  • Switching to the printer queue, printing, then back to the portal
  • A single label took roughly 2 minutes end-to-end
  • Total per-parcel packing time landed around 5 minutes

Scale that up across 50 orders a day and you've lost half a working day to platform context-switching alone.

/ The Approach

SwiftPack is a purpose-built packing terminal that consolidates all three platforms into one interface. One screen, one login, one button to print.

SYSTEM DIAGRAM Shopee + Lazada + Shopify APIs → Raspberry Pi → Touchscreen + Thermal Printer

The interface model

I'm modelling the UI after the Bambu Lab X1 Carbon 3D printer interface. It's a great reference for purpose-built embedded touchscreens: large tap targets, minimal text, status-first design, zero clutter. Built for use with gloves on, in a hurry, with one hand.

System architecture

  • Hardware: Raspberry Pi + 7" touchscreen + thermal label printer
  • Connectivity: Shopee, Lazada, Shopify seller APIs (REST)
  • State management: Local queue of pending orders, synced across platforms
  • Output: Direct-to-printer label generation, no PDF intermediary
/ Key Decisions

Print directly, skip the PDF

Most seller portals make you download a PDF, then print it. That round trip costs 30+ seconds per label. SwiftPack sends label data directly to the thermal printer — no PDF, no download, no preview screen. It prints the moment you tap "Print."

One login, persistent sessions

The biggest time-sink isn't packing — it's signing in. SwiftPack stores OAuth tokens for all three platforms and refreshes them automatically. You sign in once, weeks ago, and the system just works.

Built for a specific user (me)

I'm building this for the way I actually pack at Fragrance Castle. That means optimising for the real workflow — gloves on, hands sometimes busy, multiple parcels at once — instead of designing for an imagined average user.

/ Target Outcomes
Label printing time2 min → 30 sec (–75%)
Total packing time5 min → 1.5 min (–70%)
Platforms unifiedShopee + Lazada + Shopify in one view
Sign-ins per sessionMultiple → 0 (persistent OAuth)

At 50 orders/day, this is a recovery of roughly 3 hours of operational time per day — time that can go into customer service, new product launches, or simply ending the workday earlier.

/ Next Phase

Once the Raspberry Pi prototype is stable, I plan to:

  • Port to a cheaper microcontroller. The Pi is overkill — a Pi Zero or ESP32-based platform could halve the BOM
  • Productise for other SEA sellers. The problem isn't unique to Fragrance Castle. Most small SEA e-commerce sellers run on the same three platforms with the same bottleneck
  • Add a barcode scanner module for inventory cross-check at the moment of packing
