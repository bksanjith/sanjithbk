Parametric Lattice Design Study

A coursework exploration in Rhino + Grasshopper investigating how computational design rules translate into physical geometry and material distribution.

/ The Brief

For the Design Computing module at SUTD, I designed a parametric 3D model with adjustable density and form parameters. The goal was to understand how computational rules — not direct modelling — shape physical geometry.

Rather than designing one fixed object, the brief was to design a family of objects generated from a single set of rules. Move a slider, get a new variant.

/ The Approach

I built the system in Grasshopper as a visual node graph that takes a small set of input parameters and generates the full 3D form.

GRASSHOPPER SCRIPT SCREENSHOT The node graph driving the model

Parameter set

  • Lattice density: Controls how tightly packed the cells are
  • Form curvature: Shapes the overall envelope of the object
  • Cell scale variation: Adds organic non-uniformity to the lattice

/ What I Took Away

The biggest shift was learning to think in rules instead of geometry. Traditional CAD asks you to model what you want. Parametric design asks you to describe how it should be generated — and then explore the space of possible outputs.

This matters for product development because it lets you:

  • Test design variants quickly without rebuilding from scratch
  • Adapt designs to constraints (cost, weight, manufacturing tolerance) by changing inputs
  • Couple structural performance to form — useful for lightweighting, lattice infills, or generative design
